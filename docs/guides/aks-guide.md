# AKS Guide

> **🚧 Status: in progress — not yet written.** This guide covers an upcoming phase of the platform build and will be written as that phase is delivered. The summary below describes what it will contain.

**Purpose:** How the services are containerized and run on a managed Kubernetes (AKS) cluster — ingress, autoscaling, health management, and workload identity.

**What it will cover:** packaging each service as a container, deploying onto a managed Kubernetes cluster, and configuring ingress, horizontal autoscaling, health probes, and workload-identity-based access to cloud resources with no stored secrets.

For where this fits in the overall build, see the [Development Guide](../../DevelopmentGuide.md).
