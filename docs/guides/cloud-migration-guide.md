# Cloud Migration Guide

> **🚧 Status: in progress — not yet written.** This guide covers an upcoming phase of the platform build and will be written as that phase is delivered. The summary below describes what it will contain.

**Purpose:** How the application is migrated from local infrastructure to managed cloud services — managed data stores, messaging, identity, and secret storage — and the patterns that make the services genuinely cloud-native.

**What it will cover:** wiring each service to its managed data store, messaging backbone, identity provider, and secret store; replacing connection-string secrets with token-based authentication; and the application-code changes that make the services cloud-native.

For where this fits in the overall build, see the [Development Guide](../../DevelopmentGuide.md).
